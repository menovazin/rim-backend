# RIM (Backend+NestJS)

## Краткое описание

NestJS-зеркало публичного REST API [rickandmortyapi.com](https://rickandmortyapi.com/api) — не прокси, а самостоятельный сервер с собственной базой SQLite и локальными картинками персонажей. Стек: NestJS + SQLite (TypeORM + better-sqlite3) + Docker.

Если база пуста (например при первом запуске или чистом деплое), сервер сам скачивает персонажей, локации, эпизоды и картинки с внешнего Rick and Morty API и сохраняет их у себя. При следующих стартах обновляет уже имеющиеся записи, а не ходит в upstream на каждый запрос клиента.

## Стек технологий

| Слой | Технология | Назначение |
|---|---|---|
| Фреймворк | NestJS 10 | HTTP-сервер, контроллеры, DI |
| База данных | SQLite (better-sqlite3) | Локальное хранилище |
| ORM | TypeORM | Модели, репозитории |
| HTTP-клиент | axios | Сидер: скачивание с rickandmortyapi.com |
| Статика | @nestjs/serve-static | Раздача локальных картинок персонажей |
| Конфиг | @nestjs/config | Переменные окружения (.env) |
| Деплой | Docker + docker-compose | Контейнеризация |
| CI/CD | GitHub Actions | Автодеплой при пуше в main |

## Особенности

- **API публичный, без авторизации** — три ресурса: `/api/character`, `/api/location`, `/api/episode`. У каждого — список с пагинацией и получение по id; структура ответа совместима с rickandmortyapi.com.
- **Сидер** (см. ниже) — при запуске и каждом деплое проверяет БД и поддерживает её актуальной.
- **Локальные картинки** — ссылки на `image` указывают на локальный URL (`https://{domain}/api/character/avatar/{id}.jpeg`), а не на оригинальный rickandmortyapi.com.
- **Relative API URLs** — ссылки `next`/`prev` в `info` отдаются относительными.
- **Корневой endpoint** `/api` — discovery-ответ со ссылками на `characters`, `locations`, `episodes`.
- **CORS global** — открыт для всех origin.
- **E2E-тесты** — интеграционные тесты на API-контракт.
- **Автодеплой** — GitHub Actions: push в `main` → CI → SSH-deploy через `docker compose`.

### Seeder

При запуске (и при каждом деплое) сидер проверяет БД:

- **Если хотя бы одна из 3 таблиц пуста** — полный сид: скачивает данные и картинки персонажей с `https://rickandmortyapi.com/api`.
- **Если все таблицы заполнены** — upsert: обновляет изменившиеся записи без полного пересоздания.
- **Устойчивость к сетевым сбоям** — retry с задержками между попытками.
- **Версионирование данных** — отслеживание актуальности сид-данных.

## Запуск

**Локальная разработка:**
```bash
npm install
npm run start:dev
```

**Контейнерная разработка:**
```bash
docker compose up -d --build
```
